# Question 31

## Question
You have an Azure subscription.

Users access the resources in the subscription from either home or from customer sites. From home, users must establish a point-to-site VPN to access Azure resources. The users on the customer sites access the Azure resources by using site-to-site VPNs.

You have a line-of-business app named App1 that runs on several Azure virtual machines. The virtual machines run Windows Server 2016.

You need to ensure that the connections to App1 are spread across all the virtual machines.

What are two possible Azure services that you can use? Each correct answer presents a complete solution.

- A. an internal load balancer  
- B. a public load balancer  
- C. an Azure Content Delivery Network (CDN)  
- D. Traffic Manager  
- E. an Azure Application Gateway  

---

## Correct Answers
**A. an internal load balancer**  
**E. an Azure Application Gateway**

---

## Explanation

The requirement is to distribute incoming connections across multiple Azure VMs hosting a line-of-business application (App1).

Both of the following services can achieve load distribution:

---

### A. Internal Load Balancer (Correct)

An **Azure Internal Load Balancer (ILB)**:
- Distributes traffic within a virtual network
- Supports private IP-based access (ideal for VPN-connected users)
- Spreads connections across multiple VMs in a backend pool
- Works well for internal line-of-business applications like App1

Since users access via:
- Point-to-site VPN (home users)
- Site-to-site VPN (customer sites)

an internal load balancer is appropriate.

---

### E. Azure Application Gateway (Correct)

**Application Gateway**:
- Layer 7 load balancer (HTTP/HTTPS)
- Supports web application traffic distribution
- Provides routing, SSL termination, and WAF capabilities
- Can distribute traffic across multiple backend VMs

It is suitable when App1 is a web-based application.

---

## Why the Other Options Are Incorrect

### B. Public Load Balancer
Incorrect.

A public load balancer is used for internet-facing applications, but App1 is accessed via VPN (private network access).

---

### C. Azure Content Delivery Network (CDN)
Incorrect.

CDN is used for caching static content globally, not for distributing application traffic across backend VMs.

---

### D. Traffic Manager
Incorrect.

Traffic Manager is DNS-based routing and does not distribute traffic directly to VMs within a virtual network.

---

---

- Whether guest invitations are allowed

- Elevate access to manage all Azure resources

You have an Azure Active Directory (Azure AD) tenant named `adatum.com`.

| Name | Group Type | Membership Type | Membership Rule |

| Group1 | Security | Dynamic User | `(user.city -startsWith "m")` |

You create two user accounts configured as follows:

| Name | City | Department | Office 365 License Assigned |
|---|---|---|---|
| User1 | Montreal | Human Resources | Yes |
| User2 | Melbourne | Marketing | No |

Of which groups are User1 and User2 members?

---

## Answer

| User | Group Membership |
|---|---|
| User1 | Group1 |
| User2 | Group1 and Group2 |

---

## Explanation

### Group1 Rule
```text
(user.city -startsWith "m")
```

Both users match because:
- Montreal → starts with “M”
- Melbourne → starts with “M”

Therefore:
- User1 ∈ Group1
- User2 ∈ Group1

---

### Group2 Rule
```text
(user.department -notIn ["human resources"])
```

- User1 department = Human Resources  
  → Excluded from Group2

- User2 department = Marketing  
  → Included in Group2

Therefore:
- User1 ∉ Group2
- User2 ∈ Group2

Group3 uses **Assigned membership**, meaning users must be manually added.

- Neither User1 nor User2 are members of Group3.

# Question 43

## Question
You have an Azure subscription that contains a resource group named `TestRG`. You use `TestRG` to validate an Azure deployment.

`TestRG` contains the following resources:

| Name | Type | Details |
|---|---|---|
| VM1 | Virtual Machine | VM1 is running and configured to back up to Vault1 daily |
| Vault1 | Recovery Services Vault | Vault1 includes all backups of VM1 |
| VNET1 | Virtual Network | VNET1 has a resource lock of type DELETE |

You need to delete `TestRG`.

What should you do first?

- A. Modify the backup configurations of VM1 and modify the resource lock type of VNET1  
- B. Remove the resource lock from VNET1 and delete all data in Vault1  
- C. Turn off VM1 and remove the resource lock from VNET1  
- D. Turn off VM1 and delete all data in Vault1  

---

## Correct Answer
**B. Remove the resource lock from VNET1 and delete all data in Vault1**

---

## Explanation

A resource group cannot be deleted if:
- Resources inside it have a **Delete lock**
- A **Recovery Services Vault** still contains protected backup items or backup data

### Required actions:

#### 1. Remove the DELETE lock from VNET1
The delete lock prevents deletion of:
- The virtual network
- The resource group containing it

---

#### 2. Delete backup data from Vault1
Recovery Services Vaults cannot be deleted while:
- Backup items exist
- Soft delete/protected items remain configured

You must stop backup protection and delete backup data first.

Once these two blockers are removed, the resource group can be deleted.

---

## Why the Other Options Are Incorrect

### A. Modify backup configurations and modify lock type
Incorrect.

The backup data still exists, and merely changing the lock type is insufficient.

---

### C. Turn off VM1 and remove the lock
Incorrect.

Stopping the VM does not remove backup dependencies in the Recovery Services Vault.

---

### D. Turn off VM1 and delete backup data
Incorrect.

The DELETE lock on VNET1 would still block resource group deletion.

# Question 45

## Question
You have a registered DNS domain named `contoso.com`.

You create a public Azure DNS zone named `contoso.com`.

You need to ensure that records created in the `contoso.com` zone are resolvable from the internet.

What should you do?

- A. Create NS records in contoso.com.  
- B. Modify the SOA record in the DNS domain registrar.  
- C. Create the SOA record in contoso.com.  
- D. Modify the NS records in the DNS domain registrar.  

---

## Correct Answer
**D. Modify the NS records in the DNS domain registrar.**

---

## Explanation

Creating a public Azure DNS zone alone does not make it authoritative on the internet. You must delegate the domain from the domain registrar to Azure DNS.

To do this, you update the **Name Server (NS) records at the domain registrar** to point to Azure DNS name servers.

### What happens after this:
- Internet DNS queries for `contoso.com` go to Azure DNS
- Records in the Azure DNS zone become publicly resolvable

---

## Why the Other Options Are Incorrect

### A. Create NS records in contoso.com
Incorrect.

NS records inside Azure DNS do not delegate authority from the internet.

---

### B. Modify the SOA record in the DNS domain registrar
Incorrect.

SOA records define zone properties but do not control delegation.

---

### C. Create the SOA record in contoso.com
Incorrect.

SOA is automatically created and does not enable internet resolution.
