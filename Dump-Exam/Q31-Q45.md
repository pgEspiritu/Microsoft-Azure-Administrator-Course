# Question 31

## Question
You have an Azure subscription.

Users access the resources in the subscription from either home or from customer sites. From home, users must establish a point-to-site VPN to access Azure resources. The users on the customer sites access the Azure resources by using site-to-site VPNs.

You have a line-of-business app named App1 that runs on several Azure virtual machines. The virtual machines run Windows Server 2016.

You need to ensure that the connections to App1 are spread across all the virtual machines.

What are two possible Azure services that you can use? Each correct answer presents a complete solution.

- A. an internal load balancer  
- B. a public load balancer  
- C. an Azure Content Delivery Network (CDN)  
- D. Traffic Manager  
- E. an Azure Application Gateway  

---

## Correct Answers
**A. an internal load balancer**  
**E. an Azure Application Gateway**

---

## Explanation

The requirement is to distribute incoming connections across multiple Azure VMs hosting a line-of-business application (App1).

Both of the following services can achieve load distribution:

---

### A. Internal Load Balancer (Correct)

An **Azure Internal Load Balancer (ILB)**:
- Distributes traffic within a virtual network
- Supports private IP-based access (ideal for VPN-connected users)
- Spreads connections across multiple VMs in a backend pool
- Works well for internal line-of-business applications like App1

Since users access via:
- Point-to-site VPN (home users)
- Site-to-site VPN (customer sites)

an internal load balancer is appropriate.

---

### E. Azure Application Gateway (Correct)

**Application Gateway**:
- Layer 7 load balancer (HTTP/HTTPS)
- Supports web application traffic distribution
- Provides routing, SSL termination, and WAF capabilities
- Can distribute traffic across multiple backend VMs

It is suitable when App1 is a web-based application.

---

## Why the Other Options Are Incorrect

### B. Public Load Balancer
Incorrect.

A public load balancer is used for internet-facing applications, but App1 is accessed via VPN (private network access).

---

### C. Azure Content Delivery Network (CDN)
Incorrect.

CDN is used for caching static content globally, not for distributing application traffic across backend VMs.

---

### D. Traffic Manager
Incorrect.

Traffic Manager is DNS-based routing and does not distribute traffic directly to VMs within a virtual network.

---

# Question 32

## Question
You have an Azure subscription.

You have 100 Azure virtual machines.

You need to quickly identify underutilized virtual machines that can have their service tier changed to a less expensive offering.

Which blade should you use?

- A. Monitor  
- B. Advisor  
- C. Metrics  
- D. Customer insights  

---

## Correct Answer
**B. Advisor**

---

## Explanation

**Azure Advisor** is a built-in service that provides:
- Cost optimization recommendations
- Performance improvements
- Security best practices
- Reliability improvements

For this scenario, Azure Advisor specifically analyzes VM usage and can:
- Identify **underutilized virtual machines**
- Recommend **downsizing or resizing VM tiers**
- Suggest cost-saving actions

This makes it the best tool for identifying VMs that can be moved to cheaper SKUs.

---

## Why the Other Options Are Incorrect

### A. Monitor
Incorrect.

Azure Monitor is used for:
- Metrics collection
- Logs
- Alerts

It does not provide cost optimization recommendations.

---

### C. Metrics
Incorrect.

Metrics provide raw performance data (CPU, memory, disk usage) but do not generate optimization recommendations.

---

### D. Customer insights
Incorrect.

Customer Insights is used for customer data analytics, not infrastructure optimization.

---

# Question 33

## Question
You have an Azure Active Directory (Azure AD) tenant.

You need to create a Conditional Access policy that requires all users to use Multi-Factor Authentication when they access the Azure portal.

Which three settings should you configure?

---

## Answer

### 1. Assignments → Users
- **All users**

---

### 2. Assignments → Cloud apps or actions
- **Microsoft Azure Management (Azure Portal)**

---

### 3. Access controls → Grant
- **Grant access**
- **Require multi-factor authentication**

---

## Explanation

A Conditional Access policy in Azure AD is built using three main components:

### 1. Assignments (Who and What)
You must define:
- **Users/groups** → In this case, all users
- **Cloud apps** → Target application (Azure Portal / Azure Management)

The Azure portal is represented as:
- **Microsoft Azure Management**

---

### 2. Conditions (Optional but implied in structure)
For this scenario, no specific conditions (like location or device) are required.

---

### 3. Access Controls (What must happen)
To enforce security requirements:
- Use **Grant controls**
- Require **Multi-Factor Authentication**

---

## Summary

To meet the requirement, configure:
- All users
- Azure portal (Microsoft Azure Management)
- Require MFA under Grant control

---

# Question 34

## Question
You have an Azure Active Directory (Azure AD) tenant named contoso.onmicrosoft.com.

The User administrator role is assigned to a user named Admin1.

An external partner has a Microsoft account that uses the user1@outlook.com sign-in.

Admin1 attempts to invite the external partner to sign in to the Azure AD tenant and receives the following error message:

> "Unable to invite user user1@outlook.com. Generic authorization exception."

You need to ensure that Admin1 can invite the external partner to sign in to the Azure AD tenant.

What should you do?

- A. From the Users settings blade, modify the External collaboration settings.  
- B. From the Custom domain names blade, add a custom domain.  
- C. From the Organizational relationships blade, add an identity provider.  
- D. From the Roles and administrators blade, assign the Security administrator role to Admin1.  

---

## Correct Answer
**A. From the Users settings blade, modify the External collaboration settings.**

---

## Explanation

The issue is related to **guest user invitations** in Azure Active Directory (Azure AD).

To allow Admin1 to invite external users (such as Microsoft accounts like `user1@outlook.com`), you must configure **External collaboration settings**, which control:
- Whether guest invitations are allowed
- Who can invite guest users
- Restrictions on guest access

If these settings are too restrictive, even a User Administrator may be blocked from inviting external users, resulting in a **generic authorization exception**.

---

## Why the Other Options Are Incorrect

### B. Add a custom domain
Incorrect.

Custom domains are used for organizational branding (e.g., contoso.com), not for controlling guest invitations.

---

### C. Add an identity provider
Incorrect.

Identity providers are used for federation (e.g., Google, Facebook authentication), not for fixing guest invitation permissions in this scenario.

---

### D. Assign the Security administrator role to Admin1
Incorrect.

The Security Administrator role does not control guest invitation permissions. The issue is policy-based, not role-based.

---

# Question 35

## Question
You have an Azure subscription linked to an Azure Active Directory tenant. The tenant includes a user account named User1.

You need to ensure that User1 can assign a policy to the tenant root management group.

What should you do?

- A. Assign the Owner role for the Azure Subscription to User1, and then modify the default conditional access policies.  
- B. Assign the Owner role for the Azure subscription to User1, and then instruct User1 to configure access management for Azure resources.  
- C. Assign the Global administrator role to User1, and then instruct User1 to configure access management for Azure resources.  
- D. Create a new management group and delegate User1 as the owner of the new management group.  

---

## Correct Answer
**C. Assign the Global administrator role to User1, and then instruct User1 to configure access management for Azure resources.**

---

## Explanation

To assign policies at the **tenant root management group**, a user must have sufficient permissions at the **Azure AD tenant level**, not just subscription or resource group level.

Key requirement:
- Management group policy assignment at the root scope requires **elevated directory-level permissions**

The **Global Administrator** role in Azure AD can:
- Elevate access to manage all Azure resources
- Enable “Access management for Azure resources” (User Access Administrator at root scope)
- Assign policies at the **tenant root management group**

Without this elevation, even Subscription Owner is not enough for root management group policy assignment.

---

## Why the Other Options Are Incorrect

### A. Assign Owner role + modify conditional access policies
Incorrect.

Conditional Access has nothing to do with management group policy assignment.

---

### B. Assign Owner role + configure access management
Incorrect.

Subscription Owner alone is not sufficient for tenant root management group operations.

---

### D. Create a new management group and make User1 owner
Incorrect.

This only grants control over a new child management group, not the **tenant root management group**, which is explicitly required.
