# Question 1

## Question
Your company has several departments. Each department has a number of virtual machines (VMs).

The company has an Azure subscription that contains a resource group named `RG1`.

All VMs are located in `RG1`.

You want to associate each VM with its respective department.

What should you do?

- A. Create Azure Management Groups for each department.
- B. Create a resource group for each department.
- C. Assign tags to the virtual machines.
- D. Modify the settings of the virtual machines.

---

## Correct Answer
**C. Assign tags to the virtual machines.**

---

## Explanation

Azure **tags** are metadata labels that help organize and categorize Azure resources.

Tags consist of:
- **Name**
- **Value**

Example:
- `Department : HR`
- `Department : Finance`
- `Department : IT`

Since all virtual machines are already located in the same resource group (`RG1`), the best and simplest solution is to assign tags to identify which department each VM belongs to.

Tags are commonly used for:
- Resource organization
- Cost tracking
- Automation
- Reporting
- Governance

---

## Why the Other Options Are Incorrect

### A. Create Azure Management Groups for each department
Incorrect.

Management Groups are used to organize **multiple Azure subscriptions**, not individual virtual machines inside a resource group.

They are intended for:
- Policy management
- RBAC inheritance
- Multi-subscription governance

---

### B. Create a resource group for each department
Incorrect.

Although this could organize resources by department, the question only asks to **associate each VM with its department**, not move resources into separate groups.

Using tags is the recommended and least disruptive solution.

---

### D. Modify the settings of the virtual machines
Incorrect.

VM settings do not provide a standardized organizational method for categorizing resources by department.

Tags are specifically designed for this purpose.

# Question 5

## Question
You are planning to deploy an Ubuntu Server virtual machine to your company's Azure subscription.

You are required to implement a custom deployment that includes adding a particular trusted root certification authority (CA).

Which of the following should you use to create the virtual machine?

- A. The New-AzureRmVm cmdlet.
- B. The New-AzVM cmdlet.
- C. The Create-AzVM cmdlet.
- D. The az vm create command.

---

## Correct Answer
**D. The az vm create command.**

---

## Explanation

The `az vm create` command in Azure CLI supports custom deployments for Linux virtual machines, including Ubuntu Server.

For Linux VMs, trusted root certificates and other initialization tasks are commonly configured using:
- Cloud-init
- Custom data scripts
- Certificate injection during deployment

The Azure CLI command `az vm create` supports these custom deployment configurations directly, making it the correct choice.

Example capabilities include:
- Passing custom cloud-init files
- Configuring SSH keys
- Automating certificate installation
- Custom VM provisioning

---

## Why the Other Options Are Incorrect

### A. The New-AzureRmVm cmdlet
Incorrect.

`New-AzureRmVm` uses the deprecated AzureRM PowerShell module, which has been replaced by the Az module.

---

### B. The New-AzVM cmdlet
Incorrect.

Although `New-AzVM` can create virtual machines, the preferred method for customized Linux VM deployments involving cloud-init and certificate configuration is `az vm create`.

---

### C. The Create-AzVM cmdlet
Incorrect.

There is no valid Azure PowerShell cmdlet named `Create-AzVM`.

# Question 8

## Question
Note: This question is part of a series of questions that present the same scenario. Each question contains a unique solution that might meet the stated goals.

Your company has an Azure Active Directory (Azure AD) tenant named **weyland.com** that is configured for hybrid coexistence with the on-premises Active Directory domain.

You have a server named **DirSync1** that is configured as a DirSync server.

You create a new user account in the on-premises Active Directory. You now need to replicate the user information to Azure AD immediately.

Solution: You run the `Start-ADSyncSyncCycle -PolicyType Initial` PowerShell cmdlet.

Does the solution meet the goal?

- A. Yes  
- B. No  

---

## Correct Answer
**B. No**

---

## Explanation

The solution does **not** meet the goal.

To force immediate synchronization from on-premises Active Directory to Azure AD using Azure AD Connect, the correct command is:

- `Start-ADSyncSyncCycle -PolicyType Delta`

### Why?
- **Delta sync**: synchronizes only changes (new users, updates, deletions) → used for immediate updates
- **Initial sync**: performs a full synchronization of all objects → used for first-time setup or full resync

Since the scenario involves creating a **new user account**, only a **delta sync** is required.

---

## Why the Other Option Is Incorrect

### A. Yes
Incorrect.

`-PolicyType Initial` performs a full synchronization, which is unnecessary for a single new user and does not match the requirement of immediate replication of changes.

# Question 10

## Question
Note: This question is part of a series of questions that present the same scenario. Each question contains a unique solution that might meet the stated goals.

Your company has an Azure Active Directory (Azure AD) tenant named **weyland.com** that is configured for hybrid coexistence with the on-premises Active Directory domain.

You have a server named **DirSync1** that is configured as a DirSync server.

You create a new user account in the on-premises Active Directory. You now need to replicate the user information to Azure AD immediately.

Solution: You use **Active Directory Sites and Services** to force replication of the Global Catalog on a domain controller.

Does the solution meet the goal?

- A. Yes  
- B. No  

---

## Correct Answer
**B. No**

---

## Explanation

The solution does **not** meet the goal.

For hybrid identity environments using Azure AD Connect (DirSync), replication to Azure AD happens in this order:

1. On-premises Active Directory replication (between domain controllers)
2. Azure AD Connect sync process runs on DirSync1
3. Changes are pushed to Azure AD

Using **Active Directory Sites and Services** only forces replication **between on-premises domain controllers** (including Global Catalog updates).

It does **not** trigger synchronization from:
- On-premises AD → Azure AD

To immediately sync to Azure AD, you must trigger Azure AD Connect synchronization (e.g., `Start-ADSyncSyncCycle -PolicyType Delta`).

---

## Why the Other Option Is Incorrect

### A. Yes
Incorrect.

Forcing Global Catalog replication only updates on-premises Active Directory replication and has no direct effect on Azure AD synchronization.

# Question 12

## Question
Your company has a Microsoft Azure subscription.

The company has datacenters in Los Angeles and New York.

You are configuring the two datacenters as geo-clustered sites for site resiliency.

You need to recommend an Azure storage redundancy option.

Requirements:
- Data must be stored on multiple nodes
- Data must be stored in separate geographic locations
- Data can be read from the secondary location as well as from the primary location

Which Azure storage redundancy option should you recommend?

- A. Geo-redundant storage  
- B. Read-only geo-redundant storage  
- C. Zone-redundant storage  
- D. Locally redundant storage  

---

## Correct Answer
**B. Read-only geo-redundant storage**

---

## Explanation

**Read-only geo-redundant storage (RA-GRS)** is the correct option because it meets all the requirements:

- Stores data in a **primary region**
- Replicates data to a **secondary geographic region**
- Stores data across **multiple nodes** in both regions
- Allows **read access from the secondary region**

This satisfies both:
- High availability (geo-replication)
- Disaster recovery with read access to the secondary site

---

## Why the Other Options Are Incorrect

### A. Geo-redundant storage (GRS)
Incorrect.

GRS does replicate data to a secondary region, but:
- The secondary region is **not readable**
- It is only available after a failover

---

### C. Zone-redundant storage (ZRS)
Incorrect.

ZRS replicates data across **availability zones within the same region**, not across geographic regions.

---

### D. Locally redundant storage (LRS)
Incorrect.

LRS stores data within a **single datacenter region only**, with no geographic redundancy.

- A. Yes  
- B. No

- Review all deployed resources in JSON format
