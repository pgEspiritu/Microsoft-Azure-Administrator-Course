# Question 61

## Question
You have two Azure subscriptions named `Sub1` and `Sub2`.

An administrator creates a custom role that has an assignable scope to a resource group named `RG1` in `Sub1`.

You need to ensure that you can apply the custom role to any resource group in `Sub1` and `Sub2`. The solution must minimize administrative effort.

What should you do?

- A. Select the custom role and add Sub1 and Sub2 to the assignable scopes. Remove RG1 from the assignable scopes.  
- B. Create a new custom role for Sub1. Create a new custom role for Sub2. Remove the role from RG1.  
- C. Create a new custom role for Sub1 and add Sub2 to the assignable scopes. Remove the role from RG1.  
- D. Select the custom role and add Sub1 to the assignable scopes. Remove RG1 from the assignable scopes. Create a new custom role for Sub2.  

---

## Correct Answer
**A. Select the custom role and add Sub1 and Sub2 to the assignable scopes. Remove RG1 from the assignable scopes.**

---

## Explanation

Custom RBAC roles use:
```json
"assignableScopes"
```

to define where the role can be assigned.

Currently, the role is limited to:
```text
RG1 in Sub1
```

To allow the role to be assigned to:
- Any resource group in Sub1
- Any resource group in Sub2

you should expand the assignable scopes to:
```json
"assignableScopes": [
  "/subscriptions/Sub1",
  "/subscriptions/Sub2"
]
```

and remove the RG1-specific scope.

This:
- Enables use across both subscriptions
- Avoids creating duplicate roles
- Minimizes administrative effort

---

## Example

### Current Scope
```json
"assignableScopes": [
  "/subscriptions/Sub1/resourceGroups/RG1"
]
```

### Correct Updated Scope
```json id="az104-custom-role"
"assignableScopes": [
  "/subscriptions/Sub1",
  "/subscriptions/Sub2"
]
```

---

## Why the Other Options Are Incorrect

### B. Create separate custom roles
Incorrect.

Creates unnecessary duplicate roles and increases administration.

---

### C. Create a new role for Sub1
Incorrect.

Still creates unnecessary duplication.

---

### D. Create another role for Sub2
Incorrect.

More administrative effort than modifying the existing role.

---

# Question 62

## Question
You have an Azure Subscription that contains a storage account named `storageacct1234` and two users named `User1` and `User2`.

You assign User1 the following roles:

| Role | Scope | Group Assignment | Condition |
|---|---|---|---|
| Reader | Resource Group (inherited) | --- | None |
| Storage Blob Data Contributor | This Resource | --- | Add |

Which two actions can User1 perform?

- A. Assign roles to User2 for storageacct1234.  
- B. Upload blob data to storageacct1234.  
- C. Modify the firewall of storageacct1234.  
- D. View blob data in storageacct1234.  
- E. View file shares in storageacct1234.  

---

## Correct Answers
- **B. Upload blob data to storageacct1234**
- **D. View blob data in storageacct1234**

---

## Explanation

### ✔ Reader Role
The Reader role allows:
- Viewing Azure resource configuration
- Viewing storage account settings

But it does NOT allow:
- Modifying resources
- Uploading data
- Managing RBAC

---

### ✔ Storage Blob Data Contributor Role
The Storage Blob Data Contributor role allows:
- Read blob data
- Upload blob data
- Modify blob data
- Delete blob data

This role applies specifically to:
- Blob containers and blob data

---

## Why the Correct Answers Are Correct

### B. Upload blob data
Correct.

Storage Blob Data Contributor includes:
```text
Microsoft.Storage/storageAccounts/blobServices/containers/blobs/write
```

So User1 can upload blobs.

---

### D. View blob data
Correct.

The role also allows reading blob data.

---

## Why the Other Options Are Incorrect

### A. Assign roles to User2
Incorrect.

Assigning roles requires:
- Owner
or
- User Access Administrator

User1 does not have either role.

---

### C. Modify the firewall
Incorrect.

Firewall configuration requires management-plane permissions such as:
- Contributor
- Owner

Reader cannot modify settings.

---

### E. View file shares
Incorrect.

Storage Blob Data Contributor applies only to:
- Blob storage

It does NOT grant access to:
- Azure Files

---

# Question 63

## Question
You have an Azure subscription named `Subscription1` that contains an Azure Log Analytics workspace named `Workspace1`.

You need to view the error events from a table named `Event`.

Which query should you run in Workspace1?

- A. `select * from Event where EventType == "error"`  
- B. `Event | search "error"`  
- C. `Event | where EventType is "error"`  
- D. `Get-Event Event | where {$_.EventType == "error"}`  

---

## Correct Answer
**B. `Event | search "error"`**

---

## Explanation

Azure Log Analytics uses:
- **Kusto Query Language (KQL)**

The correct KQL syntax to search a table for the term `"error"` is:

```kusto
Event
| search "error"
```

This searches all columns in the `Event` table for the word:
```text
error
```

---

## Why the Other Options Are Incorrect

### A. SQL syntax
```sql
select * from Event where EventType == "error"
```

Incorrect.

Azure Log Analytics does NOT use SQL syntax.

---

### C. Invalid KQL syntax
```kusto
Event | where EventType is "error"
```

Incorrect.

KQL uses:
```kusto
==
```

not:
```kusto
is
```

Correct syntax would be:
```kusto
Event
| where EventType == "error"
```

---

### D. PowerShell syntax
```powershell
Get-Event Event | where {$_.EventType == "error"}
```

Incorrect.

This is PowerShell syntax, not KQL.

---

## Correct Query Example

```kusto
Event
| search "error"
```

---

# Question 66

## Question
You have an Azure subscription that is linked to an Azure AD tenant.

The tenant contains the custom role-based access control (RBAC) roles shown in the following table.

| Name | Description |
|---|---|
| Role1 | Azure Subscription Role |
| Role2 | Azure AD Role |

From the Azure portal, you need to create two custom roles named Role3 and Role4.

- Role3 will be an Azure subscription role.
- Role4 will be an Azure AD role.

Which roles can you clone to create the new roles?

---

## Answer Area

| New Role | Role to Clone |
|---|---|
| Role3 | Role1 |
| Role4 | Role2 |

---

## Explanation

Azure has two separate role systems:

### ✔ Azure RBAC Roles
Used for:
- Azure subscriptions
- Resource groups
- Azure resources

These roles can only be cloned from:
- Other Azure RBAC roles

Therefore:
- Role3 (Azure subscription role) → clone from Role1

---

### ✔ Azure AD Roles
Used for:
- Azure Active Directory management
- Users, groups, authentication, directory operations

These roles can only be cloned from:
- Other Azure AD roles

Therefore:
- Role4 (Azure AD role) → clone from Role2

---

## Final Answers

| Role | Clone From |
|---|---|
| Role3 | Role1 |
| Role4 | Role2 |

# Question 68

## Question
You have an Azure subscription that contains:
- 10 virtual machines
- A key vault named `Vault1`
- A network security group named `NSG1`

All resources are deployed to the East US Azure region.

The virtual machines are protected by using `NSG1`.

`NSG1` is configured to block all outbound traffic to the internet.

You need to ensure that the virtual machines can access `Vault1`.

The solution must:
- Use the principle of least privilege
- Minimize administrative effort

What should you configure as the destination of the outbound security rule for NSG1?

- A. an application security group  
- B. a service tag  
- C. an IP address range  

---

## Correct Answer
**B. a service tag**

---

## Explanation

Azure provides **service tags** for Azure services such as:
- AzureKeyVault
- Storage
- SQL
- AzureMonitor

A service tag:
- Represents a group of IP address prefixes managed by Azure
- Automatically updates when Azure changes service IP ranges
- Minimizes administration effort

For Azure Key Vault access, use the service tag:
```text
AzureKeyVault
```

This allows the VMs to access Vault1 without manually maintaining IP addresses.

---

## Why the Other Options Are Incorrect

### A. Application security group
Incorrect.

Application Security Groups (ASGs) organize VM network interfaces, not Azure platform services like Key Vault.

---

### C. IP address range
Incorrect.

Possible, but:
- Requires manual maintenance
- IP ranges can change
- Higher administrative effort

❌ Does not meet the requirement to minimize administrative effort.

---

## Example NSG Rule

```text
Source: VirtualNetwork
Destination: AzureKeyVault
Protocol: TCP
Port: 443
Action: Allow
```

---

## Final Answer
**B. a service tag**

---

# Question 69

## Question
You have an Azure AD tenant named `adatum.com` that contains the groups shown below.

### Groups

| Name | Member of |
|---|---|
| Group1 | None |
| Group2 | Group1 |
| Group3 | Group2 |

---

### Users

| Name | Member of |
|---|---|
| User1 | Group1 |
| User2 | Group2 |
| User3 | Group3 |
| User4 | None |

You assign the **Azure Active Directory Premium Plan 2** license to:
- Group1
- User4

Which users are assigned the license?

- A. User4 only  
- B. User1 and User4 only  
- C. User1, User2, and User4 only  
- D. User1, User2, User3, and User4  

---

## Correct Answer
**D. User1, User2, User3, and User4**

---

## Explanation

This question tests **group-based license assignment with nested groups**.

### ✔ Key Concept: Nested group licensing
When a license is assigned to a group:
- All direct members get the license
- All members of nested groups also inherit it

---

### Step-by-step breakdown

#### 1. Group1 gets the license
- User1 is directly in Group1 → ✔ licensed

Also:
- Group2 is a member of Group1 → inherited license
- Group3 is inside Group2 → also inherited

---

#### 2. License inheritance chain

- Group1 → contains User1 + Group2
- Group2 → contains User2 + Group3
- Group3 → contains User3

So:
- User1 → licensed
- User2 → licensed (via Group2 → Group1)
- User3 → licensed (via Group3 → Group2 → Group1)

---

#### 3. User4 is directly assigned
- User4 → explicitly assigned license

---

## Final Result

| User | Licensed |
|---|---|
| User1 | Yes |
| User2 | Yes |
| User3 | Yes |
| User4 | Yes |

---

## Final Answer
**D. User1, User2, User3, and User4**

---
