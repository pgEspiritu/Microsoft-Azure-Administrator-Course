# Question 46

## Question
You have an Azure subscription named `Subscription1` that contains a virtual network named `VNet1`.

You add the users in the following table:

| Name | Role |
|---|---|
| User1 | Owner |
| User2 | Security Admin |
| User3 | Network Contributor |

Which user can perform each configuration?

---

## Answer

| Configuration | User |
|---|---|
| Add a subnet to VNet1 | User3 |
| Assign a user the Reader role to VNet1 | User1 |

---

## Explanation

### Add a subnet to VNet1
To add or modify networking resources such as:
- Subnets
- Network interfaces
- Route tables

the required role is:
- **Network Contributor**

Therefore:
- **User3** can add a subnet to VNet1.

The Owner role can also do this, but the question typically expects the least privileged valid role.

---

### Assign a user the Reader role to VNet1
Assigning RBAC roles requires:
- `Microsoft.Authorization/*` permissions

The **Owner** role includes permission to:
- Assign RBAC roles
- Manage access control (IAM)

Therefore:
- **User1** can assign the Reader role.

The Network Contributor role cannot manage RBAC permissions.
The Security Admin role is related to security management, not Azure RBAC assignments.

# Question 49

## Question
You have an Azure subscription named `Sub1` that contains the following resources:

| Name | Type |
|---|---|
| RG1 | Resource Group |
| storage1 | Storage Account |
| VNet1 | Virtual Network |

You assign an Azure Policy with the following settings:

- Scope: `Sub1`
- Exclusions: `Sub1/RG1/VNet1`
- Policy definition: Append a tag and its value to resources
- Policy enforcement: Enabled
- Tag name: `Tag4`
- Tag value: `value4`

You assign tags as follows:

| Name | Tag |
|---|---|
| Sub1 | Tag1: Subscription |
| RG1 | Tag2: IT |
| storage1 | Tag3: value1 |
| VNet1 | Tag4: value2 |

For each statement, determine whether it is true.

---

## Answers

| Statement | Answer |
|---|---|
| RG1 has the Tag2: IT tag assigned only | Yes |
| storage1 has the Tag1: Subscription, Tag2: IT, Tag3: value1, and Tag4: value2 tags assigned | No |
| VNet1 has the Tag2: IT and Tag3: value2 tags assigned only | No |

---

## Explanation

### Important Concepts

#### 1. Tags are NOT inherited
Azure tags assigned to:
- Subscriptions
- Resource groups

are **not automatically inherited** by child resources.

---

#### 2. Azure Policy "Append Tag"
The policy appends:
```text
Tag4 = value4
```

to resources within scope unless excluded.

---

#### 3. VNet1 is excluded
The exclusion:
```text
Sub1/RG1/VNet1
```

means the policy does not apply to VNet1.

---

### Statement Analysis

#### Statement 1
```text
RG1 has the Tag2: IT tag assigned only
```

Correct.

RG1 only has the manually assigned:
```text
Tag2: IT
```

The append policy does not retroactively add tags to the resource group itself unless evaluated during resource operations.

Answer: **Yes**

---

#### Statement 2
```text
storage1 has Tag1, Tag2, Tag3, and Tag4:value2
```

Incorrect.

- Tags are not inherited
- storage1 only has:
  - Tag3: value1
  - Tag4: value4 (from policy)

It does NOT inherit:
- Tag1
- Tag2

Also, Tag4 value is `value4`, not `value2`.

Answer: **No**

---

#### Statement 3
```text
VNet1 has Tag2: IT and Tag3: value2 only
```

Incorrect.

VNet1 was manually assigned:
```text
Tag4: value2
```

It does not inherit Tag2 and does not have Tag3.

Answer: **No**

# Question 59

## Question
You have an Azure subscription that contains the following users in an Azure Active Directory tenant named `contoso.onmicrosoft.com`:

| Name | Role | Scope |
|---|---|---|
| User1 | Global Administrator | Azure Active Directory |
| User2 | Global Administrator | Azure Active Directory |
| User3 | User Administrator | Azure Active Directory |
| User4 | Owner | Azure Subscription |

User1 creates a new Azure Active Directory tenant named `external.contoso.onmicrosoft.com`.

You need to create new user accounts in `external.contoso.onmicrosoft.com`.

Solution: You instruct User2 to create the user accounts.

Does that meet the goal?

- A. Yes  
- B. No  

---

## Correct Answer
**B. No**

---

## Explanation

Each Azure Active Directory tenant is **independent**.

Even though User2 is a **Global Administrator in the original tenant (contoso.onmicrosoft.com)**, that role:
- Does NOT automatically grant access to other tenants
- Does NOT apply to `external.contoso.onmicrosoft.com`

To create users in the **new tenant**, User2 must first be:
- Added to the new tenant, and
- Assigned appropriate roles (e.g., Global Administrator or User Administrator) in that tenant

Since this was not done in the solution, it does not meet the goal.

---

## Why the answer is No

- Azure AD roles are **tenant-scoped**
- User2 has no permissions in `external.contoso.onmicrosoft.com` by default

---

## Final Answer
**B. No**

# Question 60

## Question
You have an Azure subscription that contains the following users in an Azure Active Directory tenant named `contoso.onmicrosoft.com`:

| Name | Role | Scope |
|---|---|---|
| User1 | Global Administrator | Azure Active Directory |
| User2 | Global Administrator | Azure Active Directory |
| User3 | User Administrator | Azure Active Directory |
| User4 | Owner | Azure Subscription |

User1 creates a new Azure Active Directory tenant named `external.contoso.onmicrosoft.com`.

You need to create new user accounts in `external.contoso.onmicrosoft.com`.

Solution: You instruct User4 to create the user accounts.

Does that meet the goal?

- A. Yes  
- B. No  

---

## Correct Answer
**B. No**

---

## Explanation

The **Owner role in an Azure subscription** is an Azure RBAC role, not an Azure AD role.

Key point:
- Azure RBAC roles (like Owner) manage **Azure resources**
- Azure AD roles (like Global Administrator or User Administrator) manage **directory objects (users, groups, tenants)**

User4:
- Has **Owner at subscription level**
- Has **no permissions in Azure AD tenant external.contoso.onmicrosoft.com**

Therefore, User4 cannot create users in the new Azure AD tenant.

To create users in the new tenant, User1/User2 would need to be assigned appropriate roles **within that tenant**.

---

## Final Answer
**B. No**
